Note12-07b · 이미지 첨부 (Cloudinary + Firestore)

슬라이드: Firebase Storage 무료 대안 ② — 외부 CDN 통합

📌 방식: 이미지를 Cloudinary에 직접 업로드(서버 없이 unsigned preset) → 받은 CDN URL을 Firestore에 저장. 이미지 본체는 Cloudinary, 메타데이터는 Firestore. 실서비스에 가까운 표준 패턴.

🎁 장점: 원본 크기 OK, 전세계 CDN, URL에 w_400,q_auto,f_auto 붙이면 자동 리사이즈·압축·WebP 변환. Cloudinary 무료: 25GB 저장 + 25GB 대역폭/월 (학기 프로젝트 100배 이상).
⚠️ config.js의 cloudinaryConfig 미설정. config.example.js의 cloudinary 발급 안내(주석) 참고 → cloudName과 uploadPreset 채우세요.

핵심 코드

// 1) Cloudinary에 직접 업로드 (서버 없음, unsigned preset)
async function uploadToCloudinary(file) {
  const fd = new FormData();
  fd.append('file', file);
  fd.append('upload_preset', UPLOAD_PRESET);  // ⭐ unsigned mode 필수

  const res = await fetch(
    `https://api.cloudinary.com/v1_1/${CLOUD_NAME}/image/upload`,
    { method: 'POST', body: fd }
  );
  if (!res.ok) throw new Error('업로드 실패');
  const data = await res.json();
  return { url: data.secure_url, publicId: data.public_id };
}

// 2) Firestore에는 URL과 publicId만 (이미지 본체 X)
await addDoc(collection(db, 'posts_cloud'), {
  text, photoURL: url, photoPublicId: publicId,
  authorId, authorName, createdAt: serverTimestamp()
});

// 3) 표시할 때 자동 리사이즈 — URL에 transformation 추가
const thumb = photoURL.replace('/upload/', '/upload/w_400,h_400,c_fill,q_auto,f_auto/');
<img src="${thumb}">

Cloudinary URL 변환 한눈에

변환 URL 조각 효과
썸네일 w_400,h_400,c_fill 400×400 정사각 크롭
품질 자동 q_auto 화질 자동 (보통 60~80%)
포맷 자동 f_auto WebP/AVIF 지원 브라우저엔 자동 변환
둥근 모서리 r_20 20px 라운드
흑백 e_grayscale 흑백 필터
합성 (워터마크) l_logo 로고 오버레이

전체 문서: cloudinary.com/documentation/image_transformations

자주 막히는 곳

  • 업로드 401/400 에러 — Upload preset이 Signed 모드인 경우. Cloudinary Console → Settings → Upload → 해당 preset → Unsigned로 변경
  • preset 이름 오타 — preset 이름은 대소문자 구분
  • CORS 에러 — 발생 거의 없음. 발생 시 Cloudinary Settings → Security → Allowed fetch domains 확인
  • Firestore permission-denied — posts_cloud 컬렉션 보안 규칙 추가 필요 (아래)
  • 이미지 안 보임 — DevTools → Network → 이미지 요청 상태 확인. 보통 URL 오타 또는 cloudName 오타
  • 삭제해도 Cloudinary에 남음 — Unsigned 모드는 *업로드만* 클라이언트 권한. 삭제는 서버측 admin API 필요. 학기 프로젝트엔 대부분 OK

보안 규칙 추가 (Firestore Rules 탭)

match /posts_cloud/{postId} {
  allow read: if request.auth != null;
  allow create: if request.auth != null
                && request.auth.uid == request.resource.data.authorId;
  allow delete: if request.auth.uid == resource.data.authorId;
}
